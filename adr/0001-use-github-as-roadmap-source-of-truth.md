# ADR-0001: GitHub をロードマップ管理の Single Source of Truth にする

- **Status**: Accepted
- **Date**: 2026-07
- **Deciders**: リポジトリ所有者

## Context

Cloud Native AI Developer になるための学習・資格・キャリア・個人開発の計画は数年にわたる長期プロジェクトであり、次の課題がある。

- 学習計画・資格情報・振り返りが、メモアプリ・ブログ・頭の中などに分散すると、更新されず陳腐化する
- 資格情報（試験の言語・廃止日・後継資格）は変化が激しく、「いつ・何を確認したか」の履歴が必要
- ChatGPT / Codex / Claude Code などの AI エージェントと協働して更新し続ける前提のため、AI が読み書きしやすい形式・場所が必要
- 方針変更（フェーズの見直し、資格の見送りなど）の経緯を後から追跡できる必要がある

## Decision

**GitHub リポジトリ（本リポジトリ）を、ロードマップ管理の唯一の情報源（Single Source of Truth）とする。**

具体的には:

- 計画・方針・記録はすべて Markdown としてこのリポジトリにコミットする
- 学習・資格・プロジェクトのタスク管理には GitHub Issue（テンプレート付き）を使う
- 変更は Pull Request またはコミットとして履歴に残す
- 大きな意思決定は ADR として `adr/` に記録する
- 他の場所（メモアプリ等）に書いた内容で残す価値のあるものは、このリポジトリに反映して初めて「決定」とみなす

## Consequences

### 良い影響

- すべての計画・記録に**バージョン履歴**が付き、変更の経緯を追跡できる
- Issue / PR / テンプレートにより、学習タスクの管理と振り返りが構造化される
- AI エージェントがリポジトリ全体を文脈として読み込み、一貫性のある提案・更新ができる
- GitHub Actions により、将来的にリンク切れチェックや定期リマインドなどの自動化が可能
- リポジトリ自体が GitHub 運用（Issue / PR / Actions）の練習台になり、GitHub 系資格の学習とも接続する

### 悪い影響・コスト

- Markdown の手入力・整形コストがかかる（→ AI エージェントの活用で軽減する）
- カレンダー・リマインダーなど時間ベースの管理は GitHub 単体では弱い（→ 必要なら外部ツールと併用するが、記録の正はこちら）
- 私的な内容（キャリアの詳細など）を書く場合、リポジトリの公開範囲に注意が必要

## Alternatives considered

| 代替案 | 不採用の理由 |
| --- | --- |
| Notion / メモアプリ | 履歴・差分管理が弱い。AI エージェント（特にコード系）との統合が GitHub より弱い |
| ブログ / Zenn 等での公開記事 | 発信には向くが、計画の継続的更新・タスク管理には不向き。SSOT ではなく成果発信先として併用する |
| ローカルファイル + クラウドストレージ | 履歴・Issue 管理・自動化がなく、長期運用で陳腐化しやすい |
| プロジェクト管理ツール (Jira / Linear 等) | 個人利用にはオーバースペック。ドキュメントと計画が分離してしまう |
