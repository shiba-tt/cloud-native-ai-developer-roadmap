# Role Definition — Cloud Native AI Developer

## 役割

クラウドネイティブな基盤の上に、AI（LLM / RAG / Agent）を組み込んだアプリケーションを、
**企画・設計・実装・運用までエンドツーエンドで担う開発者**。

モバイル（iOS)からバックエンド、データ、AI、セキュリティ、運用までを一人で貫通できることを特徴とする。

## 必要な責任範囲

| 領域 | 責任 |
| --- | --- |
| アプリケーション | モバイル / Web クライアントと API の設計・実装・テスト |
| データ | SQL / NoSQL / Vector Search を含むデータストア選定とモデリング |
| AI | RAG / Agent の設計・実装・評価（品質・コスト・レイテンシ） |
| セキュリティ | 認証・認可（OAuth / OIDC / Entra ID)、シークレット管理、基本的な脅威対策 |
| インフラ | コンテナ / Kubernetes / IaC によるプロビジョニングと構成管理 |
| デリバリー | CI/CD パイプラインの構築・運用 |
| 運用 | 監視・アラート・インシデント対応・コスト管理 |
| プロダクト | 課題定義、MVP 設計、リリース後の計測と改善 |

## できるべきこと

- 要件からアーキテクチャを設計し、トレードオフを文書（ADR）で説明できる
- iOS アプリとクラウドバックエンドを両方実装し、統合できる
- データストアを適材適所で選定し、スキーマ・パーティションを設計できる
- RAG / Agent を「デモ」ではなく「評価と運用のある機能」として実装できる
- 認証・認可を仕組みから理解し、シークレットレスな構成を設計できる
- IaC で再現可能な環境を作り、CI/CD で安全にデリバリーできる
- 障害時にログ・メトリクス・トレースから原因を特定できる
- クラウドコストを見積もり、監視し、最適化できる

## やらないこと（スコープ外）

- 大規模組織のガバナンス・コンプライアンス管理（専任領域）
- ML モデル自体の研究開発・事前学習（利用と評価に集中し、モデル開発はしない）
- 数百ノード規模のクラスタ運用のような大規模 SRE 専任業務
- Android ネイティブ開発の深掘り（必要ならクロスプラットフォームで対応）
- 「広く浅く」の資格収集（[principles.md](../docs/principles.md) 原則 6）

## iOS エンジニアからの拡張ポイント

| 既存の強み | 拡張の方向 |
| --- | --- |
| SwiftUI / クライアント実装力 | AI 機能の UX 設計・ストリーミング表示・オフライン対応へ応用 |
| API を「使う側」の経験 | API を「設計する側」へ。クライアントに優しい API 設計 |
| App Store リリース経験 | プロダクト運用・計測・改善サイクルの基盤として活用 |
| モバイルの制約への感度（通信・電池・レイテンシ） | AI アプリのコスト・レイテンシ設計へ転用 |
| Swift の型安全な設計経験 | C# / TypeScript でのバックエンド設計へ転用 |

## 将来的なキャリア候補

- **プロダクトエンジニア**: モバイル + クラウド + AI を武器に、少人数でプロダクトを作るポジション
- **テックリード / アーキテクト**: AI アプリケーションのアーキテクチャ設計をリードする
- **フリーランス / 受託**: 「モバイルから AI バックエンドまで一人で作れる」ことを強みに独立
- **個人開発者**: 自分のプロダクトを複数運用し、収益化する
- **DevRel / 技術発信**: 学習過程の発信を通じたコミュニティでのポジション構築（副次的）
