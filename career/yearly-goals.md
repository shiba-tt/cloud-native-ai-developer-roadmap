# Yearly Goals — 年次目標

年ごとの目標を管理する。年末（または年度末）に翌年分を設定し、四半期レビューで進捗を確認する。

> 以下は初期案。フェーズの進捗・公式情報の確認結果・生活状況に応じて柔軟に見直す。数字よりも「成果物が残ること」を優先する。

## Year 1（現在〜1 年後）: Foundation + Data + Security

- [ ] AZ-204 の知識を成果物（最小構成サンプル）として形にする
- [ ] Cosmos DB / データモデリングを学び、モバイルバックエンドを 1 つ設計・実装する
- [ ] OAuth / OIDC / Entra ID を理解し、認証サンプルを実装する
- [ ] DP 系・SC 系資格の公式情報を確認し、受験するか判断する（判断自体が成果）
- [ ] 統合プロダクトの題材を 1 つ選定し、MVP を作り始める
- [ ] 月次レビューを 12 回実施する

## Year 2: Cloud Native + AI Application

- [ ] Docker / Kubernetes / IaC を実務レベルにし、API を K8s にデプロイする
- [ ] CKAD / Terraform Associate の受験を判断・実行する（要公式確認のうえ）
- [ ] RAG / Agent アプリを実装し、Evaluation まで行う
- [ ] AI 系資格の日本語対応状況を確認し、受験計画を立てる
- [ ] 統合プロダクトに AI 機能を組み込みリリースする

## Year 3: Architecture & DevOps + Portfolio 完成

- [ ] AZ-305 / AZ-400 の受験を判断・実行する（要公式確認のうえ）
- [ ] 統合プロダクトを CI/CD・監視・IaC を備えた「運用されているプロダクト」にする
- [ ] ポートフォリオ（成果物一覧 + アーキテクチャ説明）を外部に公開できる状態にする
- [ ] 「iOS もできる Cloud Native AI Developer」としてキャリアの選択肢（転職・副業・フリーランス）を具体的に評価する
