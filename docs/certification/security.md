## 検討メモ

- 開発者として最優先すべきは **Entra ID / OAuth / OIDC / Managed Identity を実装レベルで理解すること**であり、資格はその確認手段
- SC-300（想定）は「アプリに認証を組み込む開発者」の観点でどこまで実用的か、試験範囲（Study guide）を見て判断する
- SC-200 / SC-100 は管理者・アーキテクト寄りの可能性が高いため、評価軸「開発者としての実用性」で慎重に判断
- 資格に関係なく、OAuth 2.0 / OIDC の仕様理解と MSAL を使った実装は Phase 3 で必ず行う
