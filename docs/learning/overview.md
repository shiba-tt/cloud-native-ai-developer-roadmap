# Learning Overview — 学習リソースの方針

学習リソース（書籍・コース・ハンズオン）の管理方針。

## ファイル構成

| ファイル | 内容 |
| --- | --- |
| [books.md](books.md) | 書籍リスト（読了状況・学びのメモ） |
| [courses.md](courses.md) | オンラインコース / Microsoft Learn のラーニングパス |
| [hands-on.md](hands-on.md) | ハンズオン課題のリスト |

## 学習の基本サイクル

1. **Input**: 公式ドキュメント / Microsoft Learn / 書籍 / コース
2. **Hands-on**: 小さな検証プロジェクトで手を動かす
3. **Output**: 成果物（リポジトリ / デプロイ / 記事）に落とし込む
4. **Log**: [templates/study-log.md](../../templates/study-log.md) で記録し、Issue に紐付ける

## リソース選定の優先順位

1. **公式ドキュメント / 公式チュートリアル** — 常に一次情報
2. **Microsoft Learn のラーニングパス** — Azure / Microsoft 資格系はここを軸にする
3. **書籍** — 体系的な理解が必要な領域（データモデリング、セキュリティ、K8s など）
4. **動画コース** — ハンズオンの流れを掴みたいとき
5. **ブログ / 記事** — 参考程度。特に試験情報・サービス仕様は鵜呑みにしない（[principles.md](../principles.md)）

## AI エージェントの学習活用

- 概念の説明・比較・演習問題の生成に活用する
- ただし、事実情報（試験・価格・サービス仕様）は必ず公式で裏取りする
- 「AI に説明してもらう」だけでなく「AI に自分が説明して添削してもらう」ことで理解度を確認する
