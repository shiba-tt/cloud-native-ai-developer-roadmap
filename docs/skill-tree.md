# Skill Tree

| カテゴリ | 必要スキル | 到達目標 | 関連資格 | 実践課題 |
|---|---|---|---|---|
| Application Development | API設計、非同期処理、エラーハンドリング、テスト | モバイル / Webから利用しやすいAPIを設計できる | AZ-204 | SwiftUIアプリとAzure Functions APIを接続する |
| Azure Platform | App Service、Functions、Storage、Messaging、Monitoring | Azure上で小規模プロダクトを構築できる | AZ-204、AZ-104、AZ-305 | 個人開発APIをAzureにデプロイする |
| Data | SQL、NoSQL、Cosmos DB、データモデリング、Vector Search | ユースケースに応じてデータストアを選択できる | DP-420、DP-300、DP-600、DP-700、DP-800 | Cosmos DBを使ったモバイルバックエンドを作る |
| AI | Azure AI、Azure OpenAI、RAG、Agent、Evaluation | AI機能を安全にアプリへ組み込める | AI-901、AI-102、AI-103、AI-200 | RAGチャットアプリを作る |
| Security | Entra ID、OAuth、OIDC、Managed Identity、Key Vault | 認証・認可とシークレット管理を設計できる | SC-300、SC-100、SC-200、SC-500 | Entra ID認証サンプルを作る |
| Cloud Native | Docker、Kubernetes、Helm、サービス設計 | コンテナ化したAPIをKubernetesへデプロイできる | KCNA、CKAD、CKA、CKS | APIをAKSまたはローカルKubernetesに配置する |
| DevOps | GitHub Actions、CI/CD、リリース戦略、品質ゲート | 継続的に安全なリリースができる | AZ-400、GitHub Actions | テスト、ビルド、デプロイのパイプラインを作る |
| Architecture | 可用性、拡張性、コスト、セキュリティ、設計レビュー | 要件からクラウド構成を説明できる | AZ-305、SC-100 | 個人開発プロダクトのアーキテクチャ図を作る |
| Observability | ログ、メトリクス、トレース、アラート | 障害調査と改善に必要な情報を設計できる | AZ-400、CNCF系 | Application Insightsで監視を設定する |
| Product Development | 課題定義、MVP、ユーザー検証、改善 | 技術学習をプロダクト価値に変換できる | なし | MVPを定義して小さく公開する |
| Mobile / iOS | Swift、SwiftUI、認証、API連携、UX | クラウドAI機能を自然なモバイル体験へ統合できる | なし | AI機能付きiOSアプリを作る |

> 注: 資格名、言語、廃止日、後継資格、更新日は受験検討時に必ず公式ページで確認する。
