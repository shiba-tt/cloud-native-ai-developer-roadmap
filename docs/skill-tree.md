# Skill Tree — スキルツリー

Cloud Native AI Developer に必要なスキルをカテゴリ別に整理する。
各カテゴリは「必要スキル / 到達目標 / 関連資格 / 実践課題」で構成する。

> 資格はすべて「検討候補」であり、受験前に公式ページで最新情報を確認する。

## 凡例

- 到達レベル: `Lv1: 知っている` → `Lv2: 使える` → `Lv3: 設計できる` → `Lv4: 教えられる/運用できる`

---

## 1. Application Development

- **必要スキル**
  - REST / gRPC API の設計と実装
  - C# (.NET) または TypeScript (Node.js) によるバックエンド開発
  - 非同期処理、エラーハンドリング、リトライ設計
  - テスト（ユニット / 統合 / E2E）
- **到達目標**: モバイルクライアントから使いやすい API を設計・実装・テストできる（Lv3）
- **関連資格**: AZ-204（取得済み）
- **実践課題**: iOS アプリ向けのバックエンド API を設計し、OpenAPI 定義とテスト付きで実装する

## 2. Azure Platform

- **必要スキル**
  - コンピューティングの選択（App Service / Functions / Container Apps / AKS）
  - Storage / Service Bus / Event Grid などのマネージドサービス
  - リソース管理（リソースグループ、タグ、RBAC）とコスト管理
- **到達目標**: 要件に応じて適切な Azure サービスを選択し、根拠を説明できる（Lv3）
- **関連資格**: AZ-204（取得済み）、AZ-104(検討)、AZ-305(検討)
- **実践課題**: 同じ API を Functions / Container Apps の 2 通りでデプロイし、コスト・運用性を比較する

## 3. Data

- **必要スキル**
  - SQL（クエリ、インデックス、実行計画の基礎）
  - Cosmos DB（パーティション設計、RU、整合性レベル、変更フィード）
  - データモデリング（リレーショナル / ドキュメント / グラフの使い分け）
  - Vector Search / Embedding の基礎
- **到達目標**: アプリ要件からデータストアを選定し、スキーマ・パーティションを設計できる（Lv3）
- **関連資格**: DP-420 など DP 系（要公式確認）
- **実践課題**: 同一ドメインを SQL と Cosmos DB の両方でモデリングし、トレードオフをまとめる

## 4. AI

- **必要スキル**
  - LLM API の利用（プロンプト設計、構造化出力、ツール呼び出し）
  - RAG（チャンキング、Embedding、検索、リランキング）
  - Agent 設計（ツール使用、状態管理、ガードレール）
  - Evaluation（品質評価、回帰テスト、コスト・レイテンシ計測）
- **到達目標**: RAG / Agent を含む AI 機能を、評価と運用まで含めて実装できる（Lv3）
- **関連資格**: AI 系資格（変更が激しいため要公式確認。日本語対応が安定してから）
- **実践課題**: 自分のドキュメントを対象にした RAG アプリを作り、評価スクリプトで品質を計測する

## 5. Security

- **必要スキル**
  - OAuth 2.0 / OIDC のフロー理解
  - Entra ID（アプリ登録、スコープ、同意）
  - Managed Identity / Key Vault によるシークレットレス構成
  - 脅威モデリングの基礎、OWASP Top 10
- **到達目標**: 認証・認可を仕組みから説明でき、シークレットレスな構成を設計できる（Lv3）
- **関連資格**: SC-300 など SC 系（要公式確認）
- **実践課題**: iOS アプリ + API を Entra ID / OIDC で保護し、Managed Identity で DB に接続する

## 6. Cloud Native

- **必要スキル**
  - Docker（イメージ設計、マルチステージビルド、セキュリティ）
  - Kubernetes（基本リソース、Helm、AKS）
  - IaC（Terraform / Bicep）
- **到達目標**: コンテナ化したアプリを IaC でプロビジョニングした環境にデプロイできる（Lv2→Lv3）
- **関連資格**: KCNA / CKAD / CKA / CKS、Terraform Associate（いずれも要公式確認）
- **実践課題**: API を Docker 化し、Terraform で構築した AKS（または Container Apps）にデプロイする

## 7. DevOps

- **必要スキル**
  - GitHub Actions による CI/CD
  - 環境分離（dev / staging / prod）とデプロイ戦略
  - テスト自動化、静的解析、セキュリティスキャン
- **到達目標**: main へのマージから本番デプロイまでを自動化したパイプラインを設計・運用できる（Lv3）
- **関連資格**: AZ-400、GitHub Actions 認定（要公式確認）
- **実践課題**: ビルド→テスト→デプロイ→スモークテストまでを含むパイプラインを構築する

## 8. Architecture

- **必要スキル**
  - 可用性・スケーラビリティ・整合性のトレードオフ理解
  - コスト見積もりと最適化
  - アーキテクチャ図とADRによる設計の言語化
- **到達目標**: 要件からアーキテクチャを設計し、トレードオフを文書で説明できる（Lv3）
- **関連資格**: AZ-305（要公式確認）
- **実践課題**: 個人開発プロダクトのアーキテクチャ図と ADR 一式を作成する

## 9. Observability

- **必要スキル**
  - 構造化ログ、メトリクス、分散トレーシング（OpenTelemetry）
  - Azure Monitor / Application Insights
  - アラート設計、SLO の基礎
- **到達目標**: 障害時に「どこで何が起きたか」を追跡できる計装を設計できる（Lv2→Lv3）
- **関連資格**: （専用資格は少ないため、AZ-400 / CNCF 系の一部として学習）
- **実践課題**: API に OpenTelemetry を計装し、ダッシュボードとアラートを設定する

## 10. Product Development

- **必要スキル**
  - 課題発見、MVP 定義、仮説検証
  - 計測（アナリティクス）と改善サイクル
  - ストア申請 / リリース運用
- **到達目標**: 企画 → MVP → リリース → 計測 → 改善のサイクルを一人で回せる（Lv3）
- **関連資格**: なし（成果物がすべて）
- **実践課題**: [docs/projects/ideas.md](projects/ideas.md) から 1 つ選び、MVP をリリースする

## 11. Mobile / iOS

- **必要スキル**（既存の強み。維持・深化する領域）
  - Swift / SwiftUI / Swift Concurrency
  - モバイルの認証（MSAL / ASWebAuthenticationSession）
  - オフライン対応、同期設計、プッシュ通知
  - クラウドバックエンドとの統合パターン
- **到達目標**: クラウド・AI 機能を組み込んだ iOS アプリを設計・実装できる（Lv4 を維持）
- **関連資格**: なし
- **実践課題**: 各フェーズの成果物に必ず iOS クライアントを用意し、強みを接続し続ける
